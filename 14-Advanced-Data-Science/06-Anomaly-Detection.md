# Anomaly Detection

## What Problem Does This Solve?

You are managing the credit card processing infrastructure for a major bank. You process 10,000 transactions per second. 

Suddenly, a user in New York buys a $5 coffee, and two minutes later, the same user's card is used to buy a $4,000 television in Paris. 

This is an **Anomaly**. It deviates so significantly from standard behavior that it arouses suspicion that it was generated by a different mechanism (in this case, fraud). 

Anomaly Detection (also known as Outlier Detection) is the process of identifying rare items, events, or observations which raise suspicions by differing significantly from the majority of the data.

---

## Why is it difficult?

1. **Extreme Class Imbalance:** In fraud detection, 99.9% of transactions are legitimate. If your ML model just predicts "Not Fraud" every single time, it will be 99.9% accurate, but completely useless to the business.
2. **Lack of Labels:** Anomalies are often unknown unknowns. Hackers invent new attacks every day. You cannot train a supervised classification model on data you have never seen. Therefore, much of anomaly detection relies on **Unsupervised Learning**.
3. **Dynamic Environments:** What is anomalous today might be normal tomorrow. (e.g., Zoom video traffic exploded during the pandemic; a static anomaly detection system would have flagged it as a DDoS attack).

---

## Core Methods

### 1. Statistical Methods (Z-Score & IQR)
The simplest approach. Calculate the mean and standard deviation of your data. If a data point is more than 3 standard deviations away from the mean (Z-Score > 3), flag it.

- **Pros:** Extremely fast. Good for sensor data (e.g., a thermometer shouldn't jump 50 degrees in one second).
- **Cons:** Assumes the data is normally distributed. Fails completely on multi-dimensional data.

### 2. Machine Learning: Isolation Forests
Standard ML algorithms try to profile the "normal" data. **Isolation Forests** do the exact opposite: they explicitly isolate the anomalies.

It builds random decision trees. Because anomalies are rare and different, it takes very few "splits" in a decision tree to isolate an anomaly into its own leaf node. Normal points, which are clustered tightly together, require many splits to isolate. 

- **Pros:** Handles high-dimensional data beautifully. Extremely fast. The industry standard for unsupervised tabular anomaly detection.
- **Cons:** Can struggle with anomalies that are clustered tightly together (collective anomalies).

### 3. Machine Learning: Local Outlier Factor (LOF)
A density-based approach. It looks at the local neighborhood of a data point. If the density around a point is much lower than the densities around its neighbors, it is flagged as an outlier.

- **Pros:** Great for finding "local" anomalies (e.g., a transaction that isn't the highest ever recorded, but is unusually high for *that specific customer*).
- **Cons:** Computationally expensive to calculate distances for all neighbors.

### 4. Deep Learning: Autoencoders
An Autoencoder is a neural network forced to compress data down to a tiny bottleneck layer, and then reconstruct it back to the original form.

You train the Autoencoder *only* on normal data. It learns to perfectly reconstruct normal transactions.
During inference, if you feed it a fraudulent transaction, the network won't know how to reconstruct it properly. The difference between the input and the output (the **Reconstruction Error**) will spike. 

- **Pros:** Can handle incredibly complex, non-linear, multi-modal data (images, text, complex network logs).
- **Cons:** Requires a massive amount of "clean" training data. Black box interpretability.

## Visualizing Anomalies

Imagine a scatter plot of data.
1. **Global Anomaly:** A single point sitting far away from the main cluster. (Z-Score catches this).
2. **Contextual Anomaly:** A point that seems normal globally, but is weird in context. (e.g., $30^\circ C$ is normal globally, but anomalous if it happens in Antarctica).
3. **Collective Anomaly:** A tight cluster of points sitting away from the main data. (e.g., A coordinated botnet attack).

## Library Implementation: Isolation Forest

```python
import numpy as np
import pandas as pd
from sklearn.ensemble import IsolationForest
import matplotlib.pyplot as plt

# Generate 500 normal transactions
normal_data = np.random.normal(loc=100, scale=10, size=(500, 2))

# Generate 20 fraudulent transactions (extreme values)
fraud_data = np.random.normal(loc=150, scale=30, size=(20, 2))

# Combine
X = np.vstack([normal_data, fraud_data])

# Train Isolation Forest
# contamination = expected percentage of anomalies (e.g., 4%)
model = IsolationForest(contamination=0.04, random_state=42)
predictions = model.fit_predict(X)

# Results: 1 for normal, -1 for anomaly
anomalies = X[predictions == -1]

print(f"Detected {len(anomalies)} anomalies out of {len(X)} total transactions.")
```

## Common Failure Cases

1. **High False Positive Rate (Alert Fatigue):** If your model flags 1,000 transactions a day as fraud, and 990 of them are false alarms, your operations team will start ignoring the alerts entirely. 
2. **Concept Drift:** As mentioned, normal behavior changes over time. Your models must be retrained continuously, or yesterday's anomaly threshold will block today's legitimate users.
3. **Adversarial Adaptation:** In fraud and cybersecurity, the anomalies are actively trying to evade detection. If you block IP addresses that send >100 requests/sec, the hackers will simply send 99 requests/sec.

---

## Industry Applications

- **Banking/Fintech:** Credit card fraud detection and Anti-Money Laundering (AML).
- **Manufacturing:** Predictive maintenance. Sensors on factory robots detect anomalous vibrations indicating a motor is about to fail, allowing repairs *before* the assembly line stops.
- **Cybersecurity:** Network intrusion detection systems identifying unusual port scanning or data exfiltration.

---

## Key Takeaways

1. Anomaly detection is largely an Unsupervised Learning problem because anomalies are rare and undefined.
2. Isolation Forests are the industry workhorse for tabular data.
3. Autoencoders use Reconstruction Error to detect anomalies in complex deep learning tasks.
4. False positives are the greatest threat to an anomaly detection system in production.
