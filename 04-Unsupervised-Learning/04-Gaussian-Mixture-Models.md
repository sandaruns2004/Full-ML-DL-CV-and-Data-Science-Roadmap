While K-Means is a powerful and simple clustering algorithm, it has two major limitations:
1.  **Hard Assignments**: A point either belongs to a cluster or it doesn't. There's no measure of uncertainty or probability.
2.  **Spherical Clusters**: It assumes all clusters are spherical and have the same variance, which fails on elongated or differently sized clusters.

**Gaussian Mixture Models (GMMs)** solve both problems by treating clustering as a probabilistic density estimation problem.

---

## 1. The Mathematics of GMM

A Gaussian Mixture Model assumes that the data points are generated from a mixture of $K$ Gaussian distributions, each with its own mean $\mu_k$ and covariance matrix $\Sigma_k$.

The probability density function evaluated at $x$ is a linear combination of $K$ Gaussian densities:

$$ p(x) = \sum_{k=1}^{K} \pi_k \mathcal{N}(x | \mu_k, \Sigma_k) $$

Where:
*   $K$ is the number of clusters (components).
*   $\pi_k$ is the **mixing coefficient** (the prior probability that a data point belongs to cluster $k$). Must satisfy $\sum_{k=1}^K \pi_k = 1$.
*   $\mathcal{N}(x | \mu_k, \Sigma_k)$ is the multivariate Gaussian probability density function for cluster $k$.

### Expectation-Maximization (EM) Algorithm

Since we don't know the assignments (latent variables), we cannot use standard Maximum Likelihood Estimation directly. We must use the **Expectation-Maximization (EM) algorithm**.

The EM algorithm iteratively refines the parameters $(\pi, \mu, \Sigma)$ in two steps:

**Step 1: E-Step (Expectation)**
Calculate the "responsibilities" $\gamma(z_{nk})$, which is the posterior probability that point $x_n$ was generated by cluster $k$, given the current parameter estimates. We use Bayes' Theorem:

$$ \gamma(z_{nk}) = \frac{\pi_k \mathcal{N}(x_n | \mu_k, \Sigma_k)}{\sum_{j=1}^{K} \pi_j \mathcal{N}(x_n | \mu_j, \Sigma_j)} $$

**Step 2: M-Step (Maximization)**
Update the parameters $(\pi, \mu, \Sigma)$ using the responsibilities calculated in the E-step to maximize the expected log-likelihood.

Let $N_k = \sum_{n=1}^{N} \gamma(z_{nk})$ be the effective number of points assigned to cluster $k$.

*   **Update Means**: $\mu_k^{new} = \frac{1}{N_k} \sum_{n=1}^{N} \gamma(z_{nk}) x_n$
*   **Update Covariances**: $\Sigma_k^{new} = \frac{1}{N_k} \sum_{n=1}^{N} \gamma(z_{nk}) (x_n - \mu_k^{new})(x_n - \mu_k^{new})^T$
*   **Update Mixing Coefficients**: $\pi_k^{new} = \frac{N_k}{N}$

We repeat the E and M steps until the log-likelihood converges.

---

## 2. Types of Covariance Matrices

One of the strengths of GMMs is the ability to restrict the shape of the clusters by choosing different types of covariance matrices:

*   **`spherical`**: Each component has its own single variance. The clusters must be spherical, but can have different sizes. (Diagonal matrix with equal elements along diagonal).
*   **`diag`** (Diagonal): Each component has its own diagonal covariance matrix. Clusters can be ellipses, but the axes must be aligned with the coordinate axes.
*   **`tied`**: All components share the exact same general covariance matrix. Clusters can have any shape, but they all must have the exact same shape and orientation.
*   **`full`**: Each component has its own general covariance matrix. Clusters can take on any elliptical shape, size, and orientation independently. (Most computationally expensive).

## 3. Implementation and Visualization

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.mixture import GaussianMixture
from sklearn.datasets import make_blobs

# Generate synthetic data with elongated clusters
n_samples = 500
np.random.seed(42)
X, y = make_blobs(n_samples=n_samples, centers=3, cluster_std=[1.0, 2.5, 0.5], random_state=42)
transformation = [[0.6, -0.6], [-0.4, 0.8]]
X_aniso = np.dot(X, transformation)

# Fit a GMM
gmm = GaussianMixture(n_components=3, covariance_type='full', random_state=42)
gmm.fit(X_aniso)

# Predict hard cluster assignments
labels = gmm.predict(X_aniso)

# Predict soft assignments (probabilities)
probs = gmm.predict_proba(X_aniso)

# Visualization
plt.figure(figsize=(8, 6))
plt.scatter(X_aniso[:, 0], X_aniso[:, 1], c=labels, s=40, cmap='viridis', zorder=2)

# Plot the GMM centers
plt.scatter(gmm.means_[:, 0], gmm.means_[:, 1], color='red', marker='X', s=100, label='Centers')

plt.title("Gaussian Mixture Model Clustering")
plt.legend()
# plt.show() # Uncomment to view
```

### Soft Clustering (Probabilities)

Notice the `predict_proba()` method above. If a point is located midway between two clusters, it might have a probability of `[0.45, 0.55, 0.0]`. This uncertainty is invaluable in real-world applications like anomaly detection or medical diagnosis.

## 4. Selecting the Number of Components (K)

Unlike K-Means where we use the Elbow Method with Inertia, GMM is a probabilistic model, so we can use information criteria to select $K$:

1.  **AIC (Akaike Information Criterion)**: Estimates the relative quality of statistical models for a given set of data. Deals with the trade-off between the goodness of fit of the model and the complexity of the model.
2.  **BIC (Bayesian Information Criterion)**: Similar to AIC but penalizes model complexity (number of parameters) more heavily than AIC.

$$ AIC = 2p - 2\ln(\hat{L}) $$
$$ BIC = p\ln(n) - 2\ln(\hat{L}) $$
*(where $p$ is the number of parameters, $n$ is number of data points, and $\hat{L}$ is the maximized value of the likelihood function).*

**Lower AIC/BIC values are better.**

```python
from sklearn.datasets import make_blobs

# Create sample dataset
X, y_true = make_blobs(n_samples=300, centers=4, cluster_std=0.60, random_state=0)

n_components = np.arange(1, 10)
models = [GaussianMixture(n, covariance_type='full', random_state=0).fit(X)
          for n in n_components]

plt.figure(figsize=(8, 5))
plt.plot(n_components, [m.bic(X_aniso) for m in models], label='BIC', marker='o')
plt.plot(n_components, [m.aic(X_aniso) for m in models], label='AIC', marker='o')
plt.legend(loc='best')
plt.xlabel('n_components')
plt.ylabel('Information Criterion')
# plt.show() # Uncomment to view
```
